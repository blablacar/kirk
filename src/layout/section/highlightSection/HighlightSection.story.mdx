## Specifications

`HighlightSection` is applied mainly on Landing Pages to highlight common rides.
It's divided on two main sections. It receives rides data and handles its display.

> _SEO_
> Content should not be completly hidden from the DOM. It uses CSS to handle visibility

### Behavior

It follows a _Grid System_: from 1 column stacked moving up to 3 equal columns.

## Components

`HighlightSection` was split in several small components: `HighlightSectionElements`

#### Title

`HighlightSection` is always inverted and has been overriden `Display1`

#### ContentItems

`HighlightSectionTitle` plus a list of `HightlightItems`

> _SEO_:
>
> - Heading is always `H2`
> - `Items` are a list `ul` of `li` elements

#### Items

Override `ItemChoice` with highlight specifity:

- Only 2 lignes,
- Text overflow is hidden
- Centered alignement;

#### Link

Action button/link for collapsing mode. It has on/off state

#### Toggle

Hide and Show content on a collapsable mode

<ArgsTable of={Toggle} />

## Usage

```jsx
import { HighlightSection } from '@blablacar/ui-library/build/layout/section/highlightSection'

export const content = [
  {heading: "Top trajets en bus", items: initialItems},
  {heading: "Top villes en bus", items: items}
]

<HighlightSection
  content={content}
  toggleLabel={{
    on: "Show more",
    off: "Show less"
  }}>
    Section content
</HighlightSection>
```
